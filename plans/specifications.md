## System Architecture

### High-Level Architecture

Bunny follows a graph-native architecture with the following layers:

```
┌─────────────────────────────────────────────────────────────┐
│                   Presentation Layer                      │
│  (Web App, Mobile Apps, Browser Extensions)          │
└──────────────────────┬──────────────────────────────────┘
                       │ GraphQL / REST
┌──────────────────────▼──────────────────────────────────┐
│                   Application Layer                      │
│  (Apollo GraphQL Server, Business Logic, Services)     │
└──────┬──────────────────────────────┬───────────────────┘
       │                              │
┌──────▼──────────┐        ┌───────▼────────────────┐
│  Cache Layer     │        │  Graph Database Layer  │
│  (Valkey)       │        │  (Neo4j Aura)        │
└───────────────────┘        └───────┬────────────────┘
                                     │
                            ┌────────▼──────────┐
                            │  Storage Layer    │
                            │  (S3, MinIO)    │
                            └───────────────────┘
```

### Component Specifications

#### Backend Services

| Service | Technology | Purpose | Scalability |

#### Relationship Types

| Relationship | From | To | Properties | Purpose |
|--------------|-------|-----|------------|---------|
| `HAS_SOURCE` | Entity | Source | - | Entity owns source |
| `POSTED_IN` | Source | Subreddit | - | Source posts to subreddit |
| `APPEARED_IN` | Media | Post | - | Media appeared in post |
| `POSTED` | User | Post | - | User created post |
| `HAS_FILTERS` | SavedBoard | FilterState | persons, sources, searchQuery | Board has filters |
| `BELONGS_TO` | Media | ImageCluster | - | Media belongs to cluster |
| `CANONICAL` | ImageCluster | Media | - | Cluster's canonical media |
| `RELATED_TO` | Entity | Entity | type, createdAt, updatedAt | Entities are related |
| `FEATURES` | Media | Entity | confidence, detectedAt | Media features entity |

### Data Normalization Rules

#### Content Normalization
All content from different sources must normalize to a consistent `Media` node structure:

- **Title**: From post title, video title, or caption
- **Source URL**: Original content URL
- **Publish Date**: ISO 8601 datetime string
- **Media Type**: Enum (IMAGE, VIDEO, TEXT, GIF)
- **Platform**: Enum (REDDIT, YOUTUBE, INSTAGRAM, TWITTER, TIKTOK)
- **Dimensions**: Width and height in pixels
- **Engagement**: Score, likes, views (platform-specific)

#### Deduplication Strategy

1. **URL Deduplication**: Bloom filter for URLs before ingestion
2. **Content Hash**: SHA-256 for text content
3. **Media Hash**: MD5 for media files
4. **Perceptual Hash**: pHash for visual similarity detection
5. **Unique Constraints**: 
   - `Post.id` (Reddit post ID)
   - `Media.sha256` (media content hash)
   - `Source.id` (source identifier)

## API Specifications

### GraphQL API

#### Endpoint
```
POST /api/graphql
Content-Type: application/json
```

#### Authentication
- **Development**: None (open access)
- **Production**: JWT tokens in Authorization header
- **Format**: `Bearer <token>`

#### Query Specifications

##### Feed Query
```graphql
query Feed($cursor: String, $limit: Int, $filters: FeedFilters) {
  feed(cursor: $cursor, limit: $limit, filters: $filters) {
    edges {
      node {
        id
        title
        imageUrl
        sourceUrl
        publishDate
        score
        width
        height
        subreddit {
          name
        }
        author {
          username
        }
        platform
        handle {
          name
          handle
        }
        mediaType
        viewCount
      }
      cursor
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}
```

**Parameters:**
- `cursor`: String (optional) - Pagination cursor (ISO datetime)
- `limit`: Int (optional, default: 20) - Items per page (max: 100)
- `filters`: FeedFilters (optional)
  - `persons`: [String!] - Entity names to filter by
  - `sources`: [String!] - Source names to filter by
  - `searchQuery`: String - Text search query

**Response:**
- `edges`: Array of feed items with cursors
- `pageInfo`: Pagination metadata
- `hasNextPage`: Boolean indicating more items available
- `endCursor`: Cursor for next page

#### Error Handling

All GraphQL responses follow this error structure:

```json
{
  "data": { ... },
  "errors": [
    {
      "message": "Error description",
      "extensions": {
        "code": "ERROR_CODE",
        "details": { ... }
      }
    }
  ]
}
```

**Error Codes:**
- `AUTHENTICATION_FAILED` - Invalid or expired token
- `AUTHORIZATION_FAILED` - Insufficient permissions
- `VALIDATION_ERROR` - Invalid input data
- `NOT_FOUND` - Resource not found
- `RATE_LIMITED` - Too many requests
- `INTERNAL_ERROR` - Server error

## Integration Specifications

### Content Source Integrations

#### Reddit Integration
- **Method**: Web scraping (no API access)
- **Rate Limiting**: Respect robots.txt, exponential backoff
- **Endpoints**:
  - Subreddit posts: `https://www.reddit.com/r/{subreddit}/hot.json`
  - Post details: `https://www.reddit.com/comments/{post_id}.json`
- **Data Extracted**: Title, URL, score, author, media URLs, created_utc
- **User Agent**: Rotating user agents to avoid blocking

#### Booru Network Integration
- **Platforms**: e621, Danbooru, Gelbooru, Yande.re, Konachan
- **Method**: REST API
- **Endpoints**: 
  - Posts: `https://{site}.net/posts.json?tags={tags}&limit={limit}`
  - Tags: `https://{site}.net/tags.json?search={query}`
- **Rate Limiting**: Respect API limits (typically 2 requests/second)
- **Data Extracted**: File URL, tags, rating, source, dimensions, artist, characters

#### RSS Feed Integration
- **Method**: Standard RSS/Atom parsing
- **Libraries**: `rss-parser` or `feedparser`
- **Data Extracted**: Title, link, published date, content, author
- **Update Frequency**: Configurable per feed (default: hourly)

#### Email Integration
- **Method**: IMAP/POP3 polling or webhook
- **Libraries**: `imapflow` or `mailparser`
- **Data Extracted**: Subject, from, body, attachments, date
- **Processing**: Extract article links from email body

### AI Service Integration

#### LiteLLM Provider
- **Purpose**: Provider abstraction for AI services
- **Models**:
  - Simple tasks: Gemini Flash ($0.075/1M tokens)
  - Summarization: Claude Haiku ($0.25/1M tokens)
  - Complex Q&A: Claude Sonnet ($3/1M tokens)
  - Vision: GPT-4V or Claude 3 Opus
- **Rate Limiting**: Implement per-model rate limits
- **Caching**: Cache responses in Valkey (24h TTL)

#### CLIP Embeddings
- **Purpose**: Visual similarity search
- **Model**: OpenAI CLIP ViT-B/32 or equivalent
- **Usage**: Generate embeddings for media thumbnails
- **Storage**: Store embeddings in Neo4j or vector database
- **Similarity Search**: Cosine similarity threshold > 0.85

### Storage Integration

#### S3-Compatible Storage
- **Providers**: AWS S3, Google Cloud Storage, Cloudflare R2, MinIO
- **Operations**:
  - Upload: `PUT /{bucket}/{path}`
  - Download: `GET /{bucket}/{path}`
  - Delete: `DELETE /{bucket}/{path}`
- **Lifecycle Policies**: 
  - Hot tier (0-30 days): Frequent access
  - Warm tier (30-90 days): Infrequent access
  - Cold tier (90+ days): Archive
- **Optimization**: 
  - Image compression (WebP format)
  - Thumbnail generation (multiple sizes)
  - Deduplication (same hash, same storage)

## Security Specifications

### Authentication & Authorization

#### Authentication Methods
- **Development**: None (open access)
- **Production**: JWT tokens with RS256 signing
- **Token Lifetime**: 7 days (refreshable)
- **Refresh Tokens**: 30 days

#### Authorization Levels
- **Read**: Access to public content
- **Write**: Modify user's own data
- **Admin**: Full system access

### Data Encryption

#### At Rest
- Database: Neo4j Aura encryption (managed)
- Storage: S3 server-side encryption (AES-256)
- Cache: Valkey encryption (optional, configurable)

#### In Transit
- API: TLS 1.3
- Database: TLS 1.3
- Storage: HTTPS only

### Data Privacy

#### Data Collection
- **Minimum**: Only necessary data for functionality
- **Opt-in**: Analytics and tracking require consent
- **Local-First**: Optional local-only mode

#### Data Retention
- **User Data**: Retained until account deletion
- **Content**: Archived after 90 days (configurable)
- **Logs**: Retained for 30 days

### Compliance

#### GDPR Compliance
- Right to access: User can export all data
- Right to rectification: User can correct data
- Right to erasure: User can delete account and data
- Right to portability: Data export in standard formats

#### CCPA Compliance
- Do not sell personal data
- Provide opt-out mechanism
- Disclose data collection practices
