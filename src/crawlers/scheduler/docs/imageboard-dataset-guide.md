# Imageboard Dataset Guide

This guide explains how to consume the imageboard datasets generated by the crawler pipeline. We provide two primary formats: the **Full Comprehensive Archive** (for local ETL and storage-heavy analysis) and the **Hugging Face Compatible Parquet** (for instant viewing and ML training).

## 1. Hugging Face Compatible Parquet
**Location:** `packs/imageboard_full_archive_YYYYMMDD/huggingface_dataset.parquet`

This is a single self-contained file (approx. 3GB) where image data is embedded as bytes.

### Loading with Python (pandas)
```python
import pandas as pd
import io
from PIL import Image

df = pd.read_parquet("huggingface_dataset.parquet")

# View a specific image
img_bytes = df.iloc[0]['image']
image = Image.open(io.BytesIO(img_bytes))
image.show()

# Access metadata
print(df.iloc[0]['text'])  # The post comment
print(df.iloc[0]['moderation_reason'])  # Why it was flagged
```

### Loading with Hugging Face `datasets`
```python
from datasets import load_dataset

# Simply upload the parquet to a HF repo and load:
dataset = load_dataset("your-username/your-repo", data_files="huggingface_dataset.parquet")

# The 'image' column will be automatically treated as a PIL object
example = dataset['train'][0]
example['image'].save("output.jpg")
```

## 2. Full Comprehensive Archive (Deduplicated)
**Location:** `packs/imageboard_full_archive_YYYYMMDD.tar.gz`

This archive separates metadata from assets to save space and is preferred for system integration.

### Structure
```text
imageboard_full_archive_YYYYMMDD/
├── dataset_index.parquet      # Metadata & file references
├── images/                    # Deduplicated library (SHA256 named)
│   ├── a1b2c3...jpg
│   └── d4e5f6...png
└── threads/                   # Raw JSON context from the crawler
    └── b/
        └── 1234567/
            └── thread.json
```

### Extraction & Usage
1. **Deduplication Logic**: The `image_sha256` column in the Parquet file corresponds to the filename in the `images/` directory. If multiple posts shared the same image, they will all point to the same file.
2. **Path Resolution**: The Parquet file contains an `image_rel_path` column (e.g., `images/a1b2...jpg`) designed to work immediately after unpacking the archive.

---

## 3. Data Schema Reference

| Column | Type | Description |
| :--- | :--- | :--- |
| `board` | string | Board name (e.g., `b`) |
| `thread_id` | string | The OP post number |
| `post_no` | int64 | The specific post number |
| `timestamp` | int64 | Unix timestamp of the post |
| `text` | string | Cleaned post comment (HTML tags removed) |
| `subject` | string | Thread subject line |
| `image_rel_path`| string | Path to image relative to archive root |
| `image_sha256` | string | SHA256 hash (used for deduplication) |
| `image` | bytes | (HF File Only) Raw image data |
| `moderated` | boolean | True if flagged as a shitpost/bot |
| `moderation_reason`| string | Reason (e.g., `bot:hex_pattern`, `low_quality:repetitive`) |

---

## 4. Moderation & Filtering
The pipeline automatically tags posts for quality. We recommend filtering the dataset before training:

```python
# Remove shitposts and bot noise
clean_df = df[df['moderated'] == False]
```

**Common Reasons:**
- `bot:hex_pattern`: Post contains long strings of random hexadecimal characters.
- `low_quality:repetitive`: Post contains common spam phrases (e.g., "nut after nut").
- `bot:keyword`: Contains specific triggers for known imageboard automated scripts.
